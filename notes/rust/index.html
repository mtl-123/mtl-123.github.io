# Rust 开发全流程 Cargo 命令手册

本文按项目创建 → 开发编码 → 代码校验 → 测试验证 → 基准测试 → 安全审计 → 版本发布 → 分发部署完整开发阶段，整理所有 Cargo 命令、配置、使用示例，新手可直接对照学习使用。

# 目录

  1. 阶段1：项目初始化
  2. 阶段2：开发编译与运行
  3. 阶段3：代码质量检查
  4. 阶段4：单元测试
  5. 阶段5：性能基准测试

# 阶段1：项目初始化

场景：从零创建 Rust 项目，生成标准项目结构。

# 核心命令

# 创建名为 project 的二进制可执行项目
cargo new project
1
2

# 生成的核心文件

  1. Cargo.toml：项目配置文件（项目信息、依赖管理）
  2. src/main.rs：程序入口文件，main 函数所在位置

# 示例

执行命令后目录结构：

project/
├── Cargo.toml  # 项目配置
└── src/
    └── main.rs # 代码入口

2

# 阶段2：开发编译与运行

场景：开发过程中编译代码、运行程序，查看效果。

# 1. 快速语法检查（不编译，速度最快）

cargo check
1
  • 作用：仅检查语法错误，不生成可执行文件，开发首选。

# 2. 编译项目

# 开发模式编译（调试信息完整，速度快）
cargo build

# 生产模式编译（优化代码，体积小、速度快）
cargo build --release
1
2
3
4
5
  • 输出文件：target/debug/项目名（开发版）、target/release/项目名（生产版）

# 3. 编译并直接运行

# 开发模式运行
cargo run

# 生产模式运行
cargo run --release
1
2
3
4
5

# 阶段3：代码质量检查

场景：统一代码风格、检查潜在bug、规范代码编写。

# 1. 自动格式化代码

cargo fmt
1
  • 作用：一键格式化所有代码，统一团队代码风格，无需手动调整缩进。

# 2. 高级代码 lint 检查

cargo clippy
1
  • 作用：比 cargo check 更严格，检查性能问题、语法错误、不规范写法，Rust 开发必备。

# 阶段4：单元测试

场景：编写测试代码，验证功能正确性。

# 核心命令

# 运行所有测试用例
cargo test

# 关闭默认测试框架（自定义测试时使用，配置在 Cargo.toml）
harness = false
1

4
5

# Cargo.toml 配置示例

[[test]]
name = "my_test"
path = "tests/test.rs"
harness = false  # 关闭 Rust 自带测试框架

4

# 测试代码示例（src/main.rs）

// 测试函数
#[test]
fn add_test() {
    assert_eq!(1 + 1, 2);
}

# 阶段5：性能基准测试

场景：测量代码运行速度，优化性能。

# 1. 安装基准测试库

在 Cargo.toml 添加开发依赖（仅测试使用，不打包进生产环境）：

[dev-dependencies]
criterion = "0.5"
1
2

# 2. 运行基准测试

cargo bench
1
  • 作用：输出代码执行时间、性能报告，用于对比优化效果。

# 阶段6：依赖安全审计

场景：检查项目第三方依赖是否存在安全漏洞。

# 1. 安装审计工具

cargo install cargo-audit
1

# 2. 执行安全检查

cargo audit
1
  • 作用：自动扫描 Cargo.toml 依赖，报告漏洞、风险等级、修复方案。

# 阶段7：自动化版本发布

工具：cargo-release 场景：一键升级版本、Git 提交、打标签、发布到开源仓库。

# 1. 安装

cargo install cargo-release
1

# 2. 核心命令

# 预览发布（不修改代码，安全测试）
cargo release patch  # 补丁版本：1.0.0 → 1.0.1
cargo release minor  # 功能版本：1.0.0 → 1.1.0
cargo release major  # 重大版本：1.0.0 → 2.0.0

# 正式执行发布
cargo release patch --execute
1

# 阶段8：自动生成更新日志

工具：git-cliff 场景：根据 Git 提交记录，自动生成结构化 CHANGELOG.md。

# 1. 安装

cargo install git-cliff
1

# 2. 核心命令

# 初始化配置文件
git-cliff --init

# 生成完整更新日志
git-cliff -o CHANGELOG.md

# 仅生成最新版本日志
git-cliff --latest -o CHANGELOG.md

# 阶段9：跨平台打包分发

工具：cargo-dist 场景：自动生成 Windows/macOS/Linux 安装包，对接 GitHub 自动发布。

# 1. 安装

cargo install cargo-dist
1

# 2. 核心命令

# 初始化项目配置
cargo dist init

# 查看发布计划
cargo dist plan

# 本地构建安装包
cargo dist build
1

# 3. 自动发布

推送 Git 标签后，GitHub Actions 自动构建全平台安装包：

git tag v1.0.0
git push --tags

# 完整开发工作流总结

# 新手标准流程（按顺序执行）

# 1. 创建项目
cargo new my_project
cd my_project

# 2. 开发编码
# 编写 src/main.rs 代码

# 3. 语法检查
cargo check

# 4. 格式化代码
cargo fmt

# 5. 规范检查
cargo clippy

# 6. 运行程序
cargo run

# 7. 测试功能
cargo test

# 8. 性能测试（可选）
cargo bench

# 9. 安全检查
cargo audit

# 10. 版本发布
cargo release patch --execute

# 11. 生成更新日志
git-cliff -o CHANGELOG.md

# 12. 打包分发
git tag v1.0.0
git push --tags
