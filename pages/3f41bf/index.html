Samba

以下是按照正常安装 Samba 的步骤，结合您提供的配置文件，编写的一份完整的部署手册。内容包括从安装到测试完成的详细步骤，并以 Markdown 格式呈现。

# Samba 部署与配置手册

本手册将指导您在 Ubuntu 系统上安装、配置和测试 Samba 服务，实现无需密码的来宾访问共享文件夹。

# 1. 安装 Samba

安装 Samba 及其依赖项：

    sudo apt install samba cifs-utils -y
    1
    // Make sure to add code blocks to your code group

    # 2. 创建共享文件夹

    # 2.1 创建共享目录

    创建一个用于共享的目录（如 /.ollama）：

    sudo mkdir -p /.ollama
    1

    # 2.2 设置权限

    设置共享目录的权限，允许所有用户读写：

    sudo chmod 777 /.ollama
sudo chown nobody:nogroup /.ollama
    1
    2

    # 3. 配置 Samba

    # 3.1 备份默认配置文件

    备份默认的 Samba 配置文件：

    sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.backup
    1

    # 3.2 编辑 Samba 配置文件

    编辑 /etc/samba/smb.conf 文件：

    sudo nano /etc/samba/smb.conf
    1

    将以下内容粘贴到文件中：

    [global]
   workgroup = WORKGROUP
   server string = %h server (Samba, Ubuntu)
   log file = /var/log/samba/log.%m
   max log size = 1000
   logging = file
   panic action = /usr/share/samba/panic-action %d
   server role = standalone server
   obey pam restrictions = yes
   unix password sync = yes
   passwd program = /usr/bin/passwd %u
   passwd chat = *Enter\snew\s*\spassword:* %n\n *Retype\snew\s*\spassword:* %n\n *password\supdated\ssuccessfully* .
   pam password change = yes
   usershare allow guests = yes
   security = user
   guest ok = yes
   guest account = nobody
   map to guest = Bad User

[PublicShare]
   path = /.ollama
   browseable = yes
   writable = yes
   guest ok = yes
   create mask = 0777
   directory mask = 0777
   force user = nobody
   force group = nogroup

    # 3.3 检查配置文件语法

    验证配置文件是否正确：

    testparm
    1

    如果输出无错误，则配置文件语法正确。

    # 4. 启动并启用 Samba 服务

    # 4.1 启动 Samba 服务

    启动 Samba 服务并设置开机自启：

    sudo systemctl restart smbd
sudo systemctl enable smbd
    1
    2

    # 4.2 检查服务状态

    确保 Samba 服务正在运行：

    sudo systemctl status smbd
    1

    # 5. 配置防火墙（如有启用）

    如果系统启用了防火墙，需开放 Samba 相关端口：

    sudo ufw allow samba
    1

    # 6. 测试 Samba 共享

    # 6.1 Windows 客户端测试

    在 Windows 资源管理器地址栏输入：

    \\服务器IP\PublicShare
    1

    应能直接访问共享文件夹，无需密码。

    # 6.2 Linux 客户端测试

    使用 smbclient 测试访问：

    smbclient //服务器IP/PublicShare -U guest
    1

    如果成功连接，说明配置正确。

    # 7. 挂载共享文件夹（Linux 客户端）

    # 7.1 创建挂载点

    在客户端创建一个挂载点：

    sudo mkdir -p /mnt/samba_share
    1

    # 7.2 挂载共享文件夹

    使用以下命令挂载共享文件夹：

    sudo mount -t cifs //服务器IP/PublicShare /mnt/samba_share -o guest,uid=1000,gid=1000
    1

    # 7.3 验证挂载

    检查挂载是否成功：

    df -h
ls /mnt/samba_share
    1
    2

    # 7.4 卸载共享文件夹

    如果不再需要挂载，可以卸载：

    sudo umount /mnt/samba_share
    1

    # 8. 自动挂载（可选）

    # 8.1 编辑 /etc/fstab

    打开 /etc/fstab 文件：

    sudo nano /etc/fstab
    1

    添加以下内容：

    //服务器IP/PublicShare  /mnt/samba_share  cifs  guest,uid=1000,gid=1000  0  0
    1

    # 8.2 测试自动挂载

    运行以下命令测试配置是否正确：

    sudo mount -a
    1

    # 9. 常见问题排查

    # 9.1 权限问题

    • 确保共享目录（如 /.ollama）及其父目录（如 /）对 nobody 用户有执行权限。

    • 使用以下命令检查权限：

      ls -ld /
ls -ld /.ollama
      1
      2

    # 9.2 日志分析

    查看 Samba 日志以定位问题：

    tail -f /var/log/samba/log.smbd
    1

    常见错误：

    • NT_STATUS_ACCESS_DENIED：通常是权限问题。
    • chdir failed：可能是路径不存在或权限不足。
