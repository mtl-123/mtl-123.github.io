# Python UV 使用手册

## 一、常用命令

```bash
# 项目初始化 [默认当前文件夹]
uv init [project_name]

# 创建虚拟环境
uv venv

# 指定 Python 版本
uv venv --python 3.11

# 激活环境
source .venv/bin/activate

# 运行脚本
uv run script.py

# 添加依赖（会更新 pyproject.toml）
uv add flask [--dev]

# 安装依赖（不更新配置文件）
uv pip install flask

# 同步项目依赖
uv sync

# 更新依赖
uv sync --upgrade

# 更新特定包
uv sync --upgrade-package flask
```

## 二、更换国内源（挑一个）

### 1. 项目配置

#### pyproject.toml 添加

```toml
[[tool.uv.index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true
```

#### uv.toml 添加

```toml
[[index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true
```

### 2. 环境变量

#### 终端临时用

```bash
uv add --default-index https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple fastmcp
```

#### 系统永久

```bash
vi ~/.bashrc
export UV_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
# 额外镜像地址
export EXTRA_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
```

## 三、安装 UV

### 1. macOS 和 Linux

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 2. Windows

在 PowerShell 中执行以下命令：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

验证安装：

```bash
uv --version
```

## 四、依赖管理

### 1. 安装依赖

```bash
uv add fastmcp
uv pip install fastmcp
```

开发环境用：`--dev`  
指定生成环境：`--production`  

```bash
uv add "numpy; python_version >= '3.11'; sys_platform == 'linux'"
```

### 2. 删除依赖

```bash
uv remove fastmcp
uv pip uninstall fastmcp
```

### 3. 查看依赖

```bash
uv pip list
```

### 4. 导出依赖

```bash
uv pip freeze > requirements.txt
uv pip compile pyproject.toml -o requirements.txt
```

### 5. 批量安装

```bash
uv pip install -r requirements.txt
uv add -r requirements.txt
uv pip install -r pyproject.toml
```

### 6. 清除缓存

```bash
uv cache clean
uv pip cache purge
```

### 7. 项目依赖树

```bash
uv tree
```

### 8. 改习惯

将以前的 `pip xxx` 替换为 `uv pip xxx`：

```bash
# 批量安装并生成锁定文件（类似 pip-tools）
uv pip compile ./requirements.in --universal --output-file ./requirements.txt

# 全局系统级别使用，添加 --system 参数
uv pip install --system pandas
```

## 五、项目管理与锁定文件

### 1. 初始化项目，管理依赖

```bash
uv init hello-world  # 初始化项目
cd hello-world

uv add 'requests==2.31.0' # 增加依赖
uv lock --upgrade-package requests # 更新项目依赖
uv remove requests # 删除项目依赖
```

### 2. 项目结构

```
.
├── .venv
│   ├── bin
│   ├── lib
│   └── pyvenv.cfg
├── .python-version
├── README.md
├── main.py
├── pyproject.toml
└── uv.lock
```

### 3. 运行项目脚本

```bash
uv run script.py
```

### 4. 更新配置项目环境

```bash
git clone git/yb-mrp.git && cd yb-mrp
uv sync # 已存在项目安装项目依赖（自动解析锁定文件）
```

### 5. 显示项目的依赖项树

```bash
uv tree --outdated # 以树形查看更新包
uv tree --depth 2  # 查看2级的依赖，防止依赖树过大
```

## 九、工作区支持

工作区通过将大型代码库拆分为具有共同依赖项的多个包来组织大型代码库。在工作区中，每个包定义自己的 `pyproject.toml`，但工作区共享一个锁定文件，确保工作区以一组一致的依赖项运行。实现 Python 版本的 Monorepo 开发模式。

要创建工作区，请将 `tool.uv.workspace` 表添加到 `pyproject.toml`，这将隐式创建以该包为根的工作区。

```toml
[project]
name = "albatross"
version = "0.1.0"
requires-python = ">=3.12"
dependencies = ["bird-feeder", "tqdm>=4,<5"]

[tool.uv.sources]
bird-feeder = { workspace = true }

[tool.uv.workspace]
members = ["packages/*"]
exclude = ["packages/seeds"]
```

工作区目录结构：

```
albatross
├── packages
│   ├── bird-feeder
│   │   ├── pyproject.toml
│   │   └── src
│   │       └── bird_feeder
│   │           ├── __init__.py
│   │           └── foo.py
│   └── seeds
│       ├── pyproject.toml
│       └── src
│           └── seeds
│               ├── __init__.py
│               └── bar.py
├── pyproject.toml
├── README.md
├── uv.lock
└── src
    └── albatross
        └── main.py
```

它们会公用一个虚拟环境和锁文件。`uv` 会处理好不同包之间的依赖关系。

运行具体的包命令：

```bash
uv run --package bird-feeder
```

## 十、全局缓存与去重

`uv` 使用缓存来避免重新下载（和重新构建）在先前运行中已经访问过的依赖项。

### 1. 查看缓存路径

```bash
uv cache dir
```

### 2. 删除所有缓存

```bash
uv cache clean
```

### 3. 删除未使用的缓存条目

```bash
uv cache purge
```

### 4. 配置缓存路径和从全局缓存中使用包的方式

```bash
$env:UV_CACHE_DIR=/path/to/code
$env:UV_LINK_MODE=hardlink
```
